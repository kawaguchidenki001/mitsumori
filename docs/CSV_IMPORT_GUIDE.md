# CSV取込フォーマットガイド

KE-Mitsumori でマスタデータを CSV ファイルから一括取込する手順とフォーマットを記載。

## 取込手順

1. 単価マスタまたは顧客マスタ画面を開く
2. 「📤 CSV取込」ボタンをクリック
3. CSV ファイルを選択
4. プレビュー画面で内容を確認（新規/更新/エラーの件数表示）
5. 「N件を取込」ボタンで実行

## 取込CSVの仕様

- **文字コード**: UTF-8（BOM 付き推奨）
- **区切り**: カンマ
- **エスケープ**: ダブルクオート `"..."` でフィールドを囲む（カンマ・改行を含む場合）
- **先頭行**: ヘッダ（列名）
- **更新判定**: 既存IDが存在する行は更新、新規IDまたは空欄なら新規作成

## 単価マスタ（unit_prices.csv）

### 必須列
| 列名 | 説明 |
|---|---|
| `item_name` | 品名（例: 電灯配線） |
| `standard_unit` | 単位（例: ケ所、m、台、式） |
| `price_embedded` | 埋込単価（円、整数） |

### 任意列
| 列名 | 説明 |
|---|---|
| `unit_price_id` | 単価ID（既存: 更新／空欄: 新規・自動採番） |
| `category_id` | 工種コード（cat01〜cat17） |
| `spec` | 仕様・型番（例: CVT38sq） |
| `price_exposed` | 露出単価（円・空欄可） |
| `maker` | メーカー（例: パナソニック） |
| `model_no` | 型番 |
| `tags` | 検索タグ（カンマ区切り） |
| `memo` | 備考 |

### 例

```csv
unit_price_id,category_id,item_name,spec,standard_unit,price_embedded,price_exposed,maker,memo
,cat05,スイッチ片切配線,,ケ所,3800,4200,パナソニック,新規追加
U022,cat05,スイッチ片切配線,,ケ所,4000,4500,パナソニック,既存U022の単価変更
,cat03,ケーブル配線,CVT60sq,m,7200,,, 
```

### エラー条件
- `item_name` 空欄
- `standard_unit` 空欄
- `price_embedded` が数値でない or マイナス
- `category_id` が存在しない工種コード

## 顧客マスタ（customers.csv）

### 必須列
| 列名 | 説明 |
|---|---|
| `company_name` | 会社名 |

### 任意列
| 列名 | 説明 |
|---|---|
| `customer_id` | 顧客ID（既存: 更新／空欄: 新規・自動採番） |
| `contact_person` | 担当者名 |
| `customer_type` | 民間／同業／元請（省略時は民間） |
| `discount_tendency` | 通常／値引き多め／値引きなし |
| `postal_code` | 郵便番号 |
| `address` | 住所 |
| `tel` | 電話 |
| `fax` | FAX |
| `email` | メール |
| `memo` | 備考 |

### 例

```csv
customer_id,company_name,contact_person,customer_type,tel,memo
,株式会社 山田工業,山田様,民間,058-XXX-XXXX,新規取引先
C001,株式会社 井上デンキ工事,井上様,民間,058-275-XXXX,既存顧客の電話変更
```

### エラー条件
- `company_name` 空欄
- `customer_type` が「民間」「同業」「元請」以外

## 出力CSVを編集して取込する手順

過去の単価データを取り込む場合：

1. 単価マスタ画面で「📥 CSV出力」をクリック → 現在のデータがCSVでダウンロード
2. Excel で開いて編集（行追加・単価変更）
3. UTF-8（BOM付き）で保存 → CSV取込

Excel で UTF-8 保存：
- 「名前を付けて保存」→ ファイルの種類「CSV UTF-8（コンマ区切り）」を選択

## バックアップ・復元（JSON形式）

CSV はマスタ単位だが、JSON バックアップは全データ（顧客・案件・明細・単価・設定）を一括保存・復元できる。

### 出力
1. ダッシュボード →「🔧 デバッグ」
2. 「📥 バックアップ出力」をクリック
3. `ke_mitsumori_backup_YYYY-MM-DD.json` がダウンロードされる

### 復元（モックモード時のみ）
1. デバッグ画面
2. 「📤 バックアップから復元」
3. JSON ファイルを選択 → 現在のモックデータを上書き

本番運用（GAS+Sheets）時のバックアップ復元は、Sheets を直接編集する方が確実なため、復元機能はモック専用としている。
